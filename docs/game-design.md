# Cosmic Match-3: дизайн и актуальное состояние

Документ фиксирует текущую игровую концепцию и поддерживается в связке с:

- `docs/game-improvements-s7.md` — запланированные улучшения;
- `docs/game-backlog.md` — приоритет задач и статус;
- `docs/game-task.md` — требования по экранным состояниям и интеграции с приложением.

## 1. Позиционирование игры

Cosmic Match — браузерный 2D match-3 на Canvas с React-обвязкой:

- быстрый вход для новых игроков;
- короткие сессии 2-6 минут;
- акцент на комбо-каскадах и целях уровня;
- рост сложности через уровни и будущие мета-механики.

## 2. Текущее ядро (реализовано)

### 2.1 Игровой цикл

- выбор хода (клик/drag/клавиатура);
- проверка валидности обмена;
- resolve-цикл: match -> clear -> fall -> refill;
- каскады и пересчет очков;
- таймер раунда и окончание матча;
- передача `onGameEnd(reason, snapshot)` в UI-слой.

### 2.2 Состояния в приложении

- `/game` — мини-лендинг игры;
- `/game/start` — подготовка матча и настройки;
- `/game/play` — активная партия;
- `/game/finish` — результат + повторный запуск/выход.

### 2.3 UI + Canvas ответственность

Через React:

- маршруты и экраны состояний;
- карточки правил/навигации;
- HUD, результат матча, ссылки на форум/лидерборд/профиль;
- сохранение последнего результата в `localStorage`.

Через Canvas:

- сетка, фишки, падение и refill;
- анимации обмена и каскадов;
- визуальные эффекты матчей.

## 3. Ближайшая эволюция (s7 roadmap)

Источник: `docs/game-improvements-s7.md`.

### 3.1 Геймплей и баланс

- переход на игру по ходам (вместо чистого таймера);
- цели уровня в формате количественных задач (не только проценты);
- расширенная система целей: цвета, блокеры, комбинированные цели;
- предсказуемая генерация поля после рестарта.

### 3.2 UX и прогрессия

- старт уровня с первого осознанного действия игрока;
- post-lose сценарий (продолжить через доп. ресурс);
- бустеры: линейные и зональные;
- мета-слой с персонажем и нарративной подачей.

### 3.3 Арт и визуальная часть

- единая палитра фишек (5 базовых цветов);
- нормализация размера фишек в клетке;
- фоновые темы уровня по прогрессу.

## 4. Правила актуализации документа

Чтобы документ было легко поддерживать:

1. Не дублировать полный список задач — хранить его в `docs/game-backlog.md`.
2. Изменения механик фиксировать кратко в разделах "Текущее ядро" и "Ближайшая эволюция".
3. При закрытии крупной задачи переносить ее:
   - из roadmap (`game-improvements-s7.md`) в "реализовано";
   - из backlog — в секцию "готово".
4. Для спорных/экспериментальных идей добавлять ссылку на issue или PR.

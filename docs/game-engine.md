# Игровой движок Match-3

## Расположение и структура 
Расположен в 'packages/client/src/game/match3/engine'. Разделен на независимые слои:
- **core** - чистая логика игрового поля и матчей;
- **bootstrap** - оркестрация игровой сессии;
- **renderer** - отрисовка на 'Canvas 2D';
- **inputController/sessionRuntime** - ввод, таймеры, подсказки;
- **matchFx** - слой VFX (частицы, вспышки и прочие спецэффекты);
- **systems** - рекорды и сервисная логика.

## Архитектура
Логика отделена от визуала:
- правила игры и resolve-пайплайн не зависят от React-компонентов;
- UI-обвязка ('Match3Screen') подписывается на 'onHudChange' и события завершения игры;
- рендер получает готовое состояние и рисует кадр без бизнес-логики.

## Ключевые модули
- 'bootstrap.ts' - точка входа 'createMatch3Game', жизненный цикл партии, API для UI.
- 'renderer.ts' - 'renderBoard', 'pickCellAt', геометрия поля, оверлеи и подсветки.
- 'resolvePass.ts' / 'resolvePipeline.ts' - расчет матчей, очистка, падение, заполнение клеток.
- 'possibleMoves.ts' - поиск возможных ходов и перетасовка при тупике.
- 'gameState.ts' / 'hudSync.ts' - модель HUD, прогресс целей, комбо, рекорды.
- 'levels.ts' - пресеты уровней: Новичок / Пилот / Асс.
- 'matchFx.ts' - эффекты матчей, премиум-комбо, анимационный цикл.

## Runtime API движка (используется UI)
- 'startPlay()', 'resetIdle()', 'destroy()';
- 'setLevel()', 'setBoardSize()', 'setDuration()';
- 'setTheme()', 'setIconTheme()', 'setVfxQuality()';
- 'setHintIdleMs()', 'setSoundEnabled()', 'setInputBlocked()'.

## Поток обработки хода
1. Ввод преобразуется в выбранную клетку/свап.
2. Проверяется валидность обмена с учетом ограничений (например, лед).
3. Выполняется swap-анимация.
4. Запускается 'resolveBoard()' в цикле проходов (до стабилизации поля).
5. Обновляются HUD и рекорды.
6. Проверяются условия завершения уровня (цель/таймер).
7. Ставится таймер подсказки на бездействие.

## Текущее состояние S7-требований
- **Есть:** уровни, цели по очкам, каскады, препятствия, VFX, выбор темы/иконок.
- **В процессе развития:** переход на режим по ходам, расширенные цели уровня, бустеры, интеграция с редактором.

## Производительность и устойчивость
- 'requestAnimationFrame' для визуальных циклов;
- ограничение количества resolve-pass для защиты от бесконечных циклов;
- отдельный 'fxCanvas', чтобы тяжелые эффекты не нагружали основной слой;
- clear/reset таймеров и обработчиков в 'destroy()'.

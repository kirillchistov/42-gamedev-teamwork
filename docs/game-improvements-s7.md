# Требования и правки по Match-3

## 1. Иконка match-3 элемента (в данном случае фишки) должна отличаться по цвету и иметь в себе 5 содержащих цветов (максимально допустимое количество цветов на одном уровне):
   - 1 цвет — синий
   - 2 цвет — зеленый
   - 3 цвет — желтый
   - 4 цвет — красный
   - 5 цвет — розовый

## 2. Игра должна вестись не на время, а на количество ходов.
   - Для выставления количества ходов должен быть реализован функционал в редакторе, считающий ходы игрока.
   - 1 действие, завершенное успехом = 1 ход.
   - Действия, которые не привели к матчу, не засчитываются ходом.

## 3. Цели должны выставляться и быть видны игроку.

### Функционал целей
#### 1. Для выставления целей в редакторе требуется отдельная плашка с визуалом, где показываются цели уровня.
#### 2. Для выставления целей в редакторе нужен функционал, где выбираются цели уровня (максимально целей может быть 4).
#### 3. Цели, которые проставляются в целях уровня, должны иметь функционал автоподсчета.
#### 4. Целями могут быть такие сущности, как:
   - фишка определенного цвета;
   - многоцветная цель, то есть все цвета;
   - блокер (ящики, прочие сущности, добавляющиеся на поле).
#### 5. В целях может выставляться:
   - 2 фишки определенного цвета;
   - все фишки;
   - блокеры.
   - Каждый тип занимает 1 ячейку целей.
   - Пример:
     - все фишки — 1 ячейка целей;
     - все фишки + ящик — 2 ячейки целей;
     - фишка определенного цвета + ящик — 2 ячейки целей.

## 4. Перемещение фишек должно происходить по принципу:
   - drag (перетягивание) с одной ячейки в другую с сопутствующей анимацией перетягивания, видимой игроку;
   - клик по ячейке -> клик на другую ячейку. Должно происходить с той же анимацией, что и перетягивание.

Текущий функционал:
- клик на ячейку -> клик на другую ячейку -> показ стрелочкой траектории.

Ожидаемый функционал:
- клик на ячейку -> клик на другую ячейку.
- Стрелочка является лишней.

## 5. Фишка должна занимать от 80% до 90% клетки, не менее.
   - В данном случае звездочки сильно выделяются, требуется реворк.
   - Луну нужно сделать больше.
   - Визуальный пример соответствия сторон — планета.

## 6. Должен быть задний фон для уровня, отдельно сгенерированный ИИ, меняющийся каждые 20 уровней.

## 7. В игре должно быть meta-menu, где игрок может видеть взаимодействия с другими персонажами, а также для игры (и её успеха) обязателен основной персонаж.
   - В данном случае можно сделать персонажа инопланетянина.
   - Чтобы персонаж полюбился юзерам, у него должна быть история (нарратив), написанная с цепляющими тейками для игрока, ведь на персонаже завязано 60% успеха игры (в том числе креативы и реклама).

## 8. На уровнях должны быть бустеры.
   - Примеры бустеров:
     1. Ракета — ломает выбранный игроком ряд по вертикали.
     2. Лазерный луч — ломает выбранный игроком ряд по горизонтали.
     3. Заморозка времени (если оставите время).
        - Если убираете время, тогда нужен бустер, который будет очищать по принципу ракета + лазерный луч,
        - либо расчищать клетку 4x4 (наносить 1 ед. урона),
        - можно интерпретировать как метеорит.

## 9. После проигрыша должно выводиться окно с предложением докупить ходы / время за просмотр рекламы (обычно +30 секунд / ходов).

## 10. Старт уровня, начало времени должно запускаться с первого матча, иначе будет большой lose rate на уровнях, ведь игроки не всегда начинают играть сразу.

## 11. Цель считается не в процентах, а отображается игроку как 0/1900 (пример).

## 12. Рандомная генерация сида после перезапуска не должна влиять на расположение блокеров на поле.

## Порядок реализации

1. Перевести core-loop на игру по ходам (добавить режим в настройку)
2. Убрать стрелку и объединить drag/click-to-click в один swap pipeline.
3. Сделать систему целей и панель целей.
4. Доработать редактор уровня: ходы, цели, блокеры.
5. Нормализовать визуалы фишек и масштаб в клетке.
6. Добавить бустеры.
7. Добавить lose modal.
8. Добавить meta-menu и персонажа.
9. Подключить фоны по группам уровней.

## Что лучше делать через React, а что через Canvas

Через **React**:
- меню;
- модалки;
- HUD;
- панель целей;
- счетчик ходов;
- редактор;
- meta-menu;
- экран поражения.

Через **Canvas**:
- сетка поля;
- фишки;
- блокеры;
- анимации перемещения;
- каскады;
- взрывы бустеров;
- подсветка совпадений.

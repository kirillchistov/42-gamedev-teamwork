# Уровни Cosmic Match

## Ключевая разница уровней (Новичок / Пилот / Асс)

### Новичок
- Для первых 5-10 матчей, чтобы привыкнуть к механике каскадов.
- Цель по очкам ниже, поле компактнее, меньше перегрузки по решениям.
- Базовый ориентир по лимиту: `50` ходов.

### Пилот
- Средняя сложность: выше плотность комбинаций и выше требование по очкам.
- Лучше всего подходит после стабильных побед на Новичке.
- Базовый ориентир по лимиту: `75` ходов.

### Асс
- Высокая сложность: больше поле, больше вариативность и цена ошибки.
- Требует планирования цепочек и осознанной игры под цель.
- Базовый ориентир по лимиту: `100` ходов.

## Как выбрать лимит ходов

На старте проекта практичнее использовать фиксированные пресеты (`50/75/100`), а не вычислять лимит формулой. Причина: пока мало статистики по реальному win-rate.

Когда накопится телеметрия (хотя бы 200-300 игр на каждый уровень), можно перейти к адаптивной формуле:

- базовый лимит по уровню;
- `+5` ходов, если win-rate за последние N игр < 35%;
- `-5` ходов, если win-rate > 70%;
- ограничение диапазона: `40..120`.

Такой подход сохраняет предсказуемость для игрока и постепенно балансирует сложность без резких скачков.

## Дорожная карта сложности игрока

### Этап 1. Вход (0-10 игр)
- Цель: научиться собирать стабильные матчи 3+ и видеть простые комбо.
- Режим: Новичок, лимит 50, умеренная цель по очкам.

### Этап 2. Стабилизация (10-30 игр)
- Цель: регулярно закрывать цель и улучшать итоговый счёт.
- Режим: Новичок / Пилот (переход после 3-5 побед подряд).

### Этап 3. Рост мастерства (30-80 игр)
- Цель: контролировать каскады, избегать пустых ходов, играть под тайминг/ходы.
- Режим: Пилот как основной, периодические заходы на Асс.

### Этап 4. Продвинутая игра (80+ игр)
- Цель: устойчивые победы на Асс и конкуренция за рекорды.
- Режим: Асс, работа над эффективностью хода и длинными комбо.

## Почему можно проиграть даже на Новичке

Чаще всего проигрыш связан с комбинацией факторов:

- игрок делает много ходов с низкой ценностью (без подготовки каскада);
- цель по очкам не отслеживается в течение матча, а только в конце;
- серия "хороших" матчей начинается слишком поздно;
- в конце партии не хватает 1-2 сильных комбо для добора цели.

Для UX лучше всегда показывать цель в явном виде (`текущий счёт / цель`) и оставшийся ресурс (`ходы` или `время`), чтобы причина поражения была понятна сразу во время игры, а не только на финальном экране.

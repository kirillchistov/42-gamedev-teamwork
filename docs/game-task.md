# Задание на разработку веб‑приложения «Игра»

В этом модуле вам предстоит разработать браузерную игру.  
Она будет состоять из двух частей: самой игры и «коммерческой» части (авторизация, таблицы лидеров и т.п.).

## Технологии

Языки TypeScript и JavaScript (версия ≥ 3.8).

- Препроцессоры или PostCSS — стилизация приложения.
- Webpack (≥ 4) — сборка приложения.
- Web API — расширенная функциональность (уведомления, запросы, fullscreen и т.п.).
- React (≥ 16.8) — UI.
- Redux — локальное хранилище данных.
- React Router — работа с History API в React.
- Node.js (≥ 12) — dev‑сервер, версия в `.nvmrc` или `engines` в `package.json`.
- Docker — эксплуатация приложения через контейнеризацию.
- ESLint, Stylelint, `.editorconfig` — единый стиль кода.
- Jest — тесты.
- Yandex Cloud — инфраструктура и вычислительные ресурсы.
- Web Workers — фоновые скрипты, офлайн‑работа, уведомления.
- Canvas API — 2D‑графика в браузере.
- Server Side Rendering (React Apps) — генерация страниц на Node.js.
- OAuth — авторизация через сторонний сервис (например, Яндекс).
- Nginx — веб‑сервер.
- CI/CD — автоматизация сборки, тестов и выката.
- SSL — безопасное соединение.
- Proxy — промежуточный сервер между клиентом и интернетом.
- PostgreSQL (≥ 11) — реляционная БД.
- MongoDB — NoSQL БД.

### Библиотеки

- Клиент: любые, кроме части с самой игрой — она пишется на «чистых» TypeScript + Canvas API + статика.
- Сервер: любые библиотеки.

## Задание на разработку

Команда самостоятельно придумывает дизайн и механику игры, иллюстрации в документе — только для вдохновения.

### Экраны приложения

Дизайн ограничен только вашим воображением: можно ориентироваться на аналоги/референсы.

1. **Страница авторизации**.  
   Поля: логин, пароль. Есть ссылка на страницу регистрации.  
   - При успешной авторизации или попытке зайти авторизованному пользователю — редирект на страницу с игрой.  
   - При ошибке выводятся сообщения об ошибках.  
   - Макет: `./media/image1.png`.

2. **Страница регистрации**.  
   Поля: имя, фамилия, почта, телефон, логин, пароль.  
   - При успехе — переход к игре.  
   - При ошибках — подсветка полей, не прошедших валидацию.  
   - Макет: `./media/image2.png`.

3. **Страница профиля**.  
   Содержит изменяемую информацию о пользователе: имя, фамилия, никнейм, почта, телефон, пароль, аватар (загрузка/удаление).  
   - Макет: `./media/image3.png`.

4. **Страница(ы) с игрой**.  
   Включают все состояния игры:
   - предзапуск (выбор персонажа, «рубашки» карты, прелоадер и пр.);
   - процесс игры;
   - завершение (результаты, кнопки «играть заново», «в главное меню»).  
   - Макет: `./media/image4.png`.

5. **Страница с лидербордом**.  
   Таблица пользователей сервиса c сортировкой по любому полю.  
   - Колонки определяет команда.  
   - Данные хранятся в БД, взаимодействие — через API.  
   - Макет: `./media/image5.png`.

6. **Страница форума**.  
   Список тем форума, возможность оставлять комментарии и ответы (в том числе с Emoji), создавать новые темы.  
   - «Ручки» форума пишутся самостоятельно.  
   - Макет: `./media/image6.png`.

7. **Страница 404**.  
   Сообщение о том, что пользователь заблудился, и кнопка возвращения в сервис.  
   - Макет: `./media/image7.png`.

8. **Страница 5xx‑ошибок**.  
   Вежливое сообщение о том, что сервис временно сломан, но проблема уже устраняется.  
   - Макет: `./media/image8.png`.

## Разработка игры

- Игра должна быть **2D**.
- На одном устройстве могут играть от 1 до 16 человек (поддержка > 1 игрока опциональна, но обсуждается с ментором).

Технологии непосредственно игры:

1. TypeScript.  
2. Canvas API.

### Обязательные фичи игры

1. **Интерфейс управления процессом**:  
   Отображение жизней, времени, очков, имён персонажей, количества карт в руке и т.п. (зависит от вашей игры).

2. **Обработка начала и конца игры**:  
   - Показ победителя и счёта.  
   - Кнопки выхода в начальный интерфейс или перезапуска игры.

3. **Превью/прелоадер**:  
   - Выбор персонажа, скина/«рубашки» или хотя бы загрузочный экран.

## Требования к проекту

- Все экраны свёрстаны и работают корректно.
- Проект на React с Redux и React Router.
- Следование практикам SOLID, DRY, KISS.
- Защита от XSS и DOS.
- Каждая страница — отдельный роут, при обновлении остаёмся на том же роуте.
- Подключена HTTP‑часть API сервиса.
- Игра корректно работает на десктопе, есть все состояния.
- ≥ 95% кода на TypeScript.
- Приложение работает без интернета (PWA/Service Worker).
- Есть хотя бы один собственный HOC и один собственный Hook.
- Обработка ошибок клиента и сервера, нет «белых экранов».
- Используются Canvas API, Fetch API и ещё ≥ 4 Web API (например, LocalStorage, Notifications).
- Lighthouse ≥ 50% по всем метрикам.
- Базовые компоненты покрыты тестами.

### Алгоритмы и структуры данных

- Собственный метод сортировки (в прототипе массивов/объектов, без `Array.prototype.sort`).
- Реализация двух структур данных: очередь и стек (нельзя использовать массивы для этих задач).

### Бэкенд/данные

- Использование PostgreSQL + Sequelize для темизации.
- Эндпоинты, требующие авторизации, закрыты от неавторизованных пользователей.
- Версия Node.js в `.nvmrc` или `engines` (не ниже 12).

Приведён пример для трёх разработчиков (можно перераспределять, согласовав с ментором).

- Redux‑хранилище.
- Thunk/Saga для запросов (кроме авторизации).
- Дополнительное Web API.

- Реализация функциональности форума (комментарии, ответы).
- Управление топиками форума на клиенте.

### Второй разработчик

- Главная страница и регистрация.
- API регистрации.
- React Router.

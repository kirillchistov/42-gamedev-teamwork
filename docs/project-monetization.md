# Project Monetization (Concept, v1)

Документ фиксирует концепцию монетизации `Cosmic Match-3` с приоритетом на не-рекламные модели.  
Реклама рассматривается как вторичный канал (компенсация, ретеншн), а не основной источник дохода.

## 1) Цели монетизации

1. Сохранить fair-play и ощущение "skill first".
2. Монетизировать комфорт и персонализацию, а не "обязательную победу за деньги".
3. Поддержать две стратегии игрока:
   - "играю бесплатно, накапливаю";
   - "плачу за удобство, скорость и кастомизацию".
4. Не ломать короткие сессии 2-6 минут и core-loop match-3.

## 2) Базовая экономика

Рекомендуется 3-валютная модель:

- **Soft currency (`credits`)**  
  Основная бесплатная валюта. Источники: уровни, квесты, дейлики, события.  
  Траты: базовые бустеры, продолжение после проигрыша, паузы, часть косметики.

- **Hard currency (`crystals`)**  
  Премиум-валюта за реальные деньги (и малые порции в ивентах).  
  Траты: быстрые продолжения, редкая косметика, battle pass, клановые слоты, эксклюзивные эффекты.

- **Energy/Hearts (`energy`)** *(опционально на этапе 2)*  
  Ограничитель сессий для midcore-ритма. Сейчас можно отложить, чтобы не душить retention в раннем продукте.

Реклама (`rewarded`) остаётся как альтернативный способ получения части ресурсов.

## 3) Набор стартовых идей

Ниже - предлагаемые драфты идей с фокусом на игровой баланс и с учетом мировых практик монетизации игр.

### 3.1 Гештальт после проигрыша

Концепция:
- после проигрыша предлагать **один из пакетов**:
  - `+5 ходов` (основной для move-mode),
  - `+20 сек` (для time-mode),
  - "мини-спас" (`+3 хода + 1 случайный бустер`) как премиум.
- каналы оплаты:
  - `credits` (дешево, но с лимитом в день),
  - `crystals` (без дневного лимита),
  - `rewarded video` (1-2 раза в день на игрока).
- цена continue должна расти на повторе в рамках одной попытки (anti-abuse).

Зачем: почти стандарт в жанре (King/Playrix-паттерн), но с контролем pay-to-win эффекта.

### 3.2 Хинты, undo, slow-mo

Концепция:
- монетизировать **QoL-функции**:
  - `Auto-hint` (умная подсказка без ожидания),
  - `Undo 1 move` (ограниченно, отмена хода),
  - `Slow-mode` в time-mode.

Зачем: позволяет набрать скилл без нервотрепки.

### 3.3 Дополнительные бустеры

Концепция:
- разделить бустеры на 3 слоя:
  - **Pre-level boosters** (до старта): ракета, бомба, color start;
  - **In-level tactical** (в процессе): молоток/перестановка/заморозка таймера;
  - **Meta boosters** (из кланов и событий): временные глобальные баффы.
- ключевой принцип: бустер помогает пройти "сложный пик", но не заменяет игру.
- ввести weekly cap на самые сильные бустеры, чтобы защитить экономику.

Зачем: позволяет избежать фрустрации "не везет с раздачей"

### 3.4 Генерация своего дизайна

Концепция:
- сделать это **главной non-ads монетизацией**:
  - скины персонажа (tiers: common/rare/epic),
  - темы поля (фон, рамки, плитки),
  - анимации матчей и финальный финишер эффекты,
  - аватар/рамка профиля/табличка в лидерборде.
- добавить "Creator Pack":
  - игрок собирает шаблон из ассетов (цвет, паттерн, части костюма),
  - публикация в клане/друзьям (после модерации/шаблонных ограничений).

Зачем: косметика монетизирует статус и самовыражение без влияния на competitive fairness.

### 3.5 Создание клана

Концепция:
- разделить on-ramp:
  - `join clan` бесплатно,
  - `create clan` за `credits` или `crystals` (с кулдауном на роспуск/создание).
- монетизировать **клановый комфорт**, а не "обязательную силу":
  - доп. слот клан-квеста,
  - косметика клана (эмблема, баннер),
  - ускорение кланового прогресса (лимитировано),
  - сезонный "Clan Pass".

Зачем: социальный слой хорошо держит retention и LTV, но требует аккуратного anti-paywall.

## 4) Дополнения (рекомендуемые каналы non-ads)

### 4.1 Starter Pack (FTUE-офер / UX новичка)

Одноразовый набор на 24-72 часа после первого дня:
- небольшое число `crystals`,
- 2-3 бустера,
- эксклюзивная косметика "Founders".

Цель: ранняя конверсия без давления.

### 4.2 Season Pass (30 дней)

Две дорожки:
- free track (софт-награды),
- premium track (косметика, умеренные бустеры, небольшой `crystals` cashback).

Цель: предсказуемая выручка + длинный цикл вовлечения.

### 4.3 Limited-time Events

- тематические ивенты с отдельной прогрессией;
- пропуск события + косметические трофеи;
- клановые ивенты с milestone-наградами.

### 4.4 Offer Wall внутри игры (без внешней рекламы)

Внутриигровой магазин-страница с персональными пакетами:
- "почти прошёл уровень" пакет;
- "вернулся после 3+ дней" reactivation pack;
- "достиг milestone" celebration pack.

## 5) Матрица "ценность -> валюта"

- **Power / прохождение**: сначала `credits`, затем `crystals`, затем rewarded как fallback.
- **Cosmetics / самовыражение**: в основном `crystals`, иногда события/pass.
- **Social / clan features**: смешанная модель (`credits` + premium upgrade).
- **Convenience / QoL**: подписка или небольшие premium-платежи.

## 6) Ограничения и anti-pay-to-win

1. Нет "обязательных платежей" для прохождения основного контента.
2. Сильные бустеры ограничены по частоте применения.
3. PvP/соревновательные рейтинги (если будут) - без premium-усилений в матче.
4. Прогресс free-игрока должен быть стабильным, пусть и медленнее.
5. Любой rewarded-механизм имеет дневной кап.

## 7) KPI для проверки гипотез

Продуктовые:
- D1 / D7 / D30 retention
- average session length
- level fail rate (особенно на пиках сложности)

Монетизация:
- conversion to payer
- ARPDAU
- ARPPU
- payer retention (D30)
- доля выручки: IAP vs rewarded

Экономика:
- inflow/outflow `credits` и `crystals`
- usage rate по бустерам
- continue rate после поражения

## 8) Поэтапный запуск

### Phase A (MVP monetization)
- `credits` + `crystals`
- continue after lose (`+5 moves`) за `credits/crystals/rewarded`
- базовый магазин бустеров
- 1-2 косметических набора

### Phase B
- starter pack
- season pass (упрощенный)
- клан: создание + базовые квесты + клановая косметика

### Phase C
- live-ops офферы (персонализация)
- creator cosmetics (шаблонный конструктор)
- ивенты + клановые сезоны

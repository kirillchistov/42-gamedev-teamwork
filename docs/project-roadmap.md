# Дорожная карта проекта

Документ основан на текущем состоянии игры и требованиях из 'docs/game-improvements-s7.md'.

## Принципы roadmap
- Приоритет у изменений, влияющих на core-геймплей и удержание.
- Каждая крупная задача должна иметь измеримый критерий приемки.
- Все изменения геймплея синхронизируются с редактором уровней и HUD.

## Этап 1. Core-геймплей (высокий приоритет)
- [ ] Переключить основной режим с таймера на **игру по ходам**.
- [ ] Считать ход только при успешном действии игрока.
- [ ] Убрать визуальную стрелку swap, оставить drag/click с единой анимацией.
- [ ] Запускать активную фазу уровня с первого результативного действия.
- [ ] Стабилизировать генерацию/рестарт (seed не ломает блокеры).

**Критерий готовности:** полный цикл партии по ходам работает без регрессий в каскадах, HUD и экране результата.

## Этап 2. Цели и редактор уровней
- [ ] Добавить панель целей в интерфейс игрока.
- [ ] Поддержать до 4 целей на уровень.
- [ ] Типы целей:
  - фишки выбранного цвета,
  - цель по всем цветам,
  - блокеры/препятствия.
- [ ] Отображение прогресса в формате 'current / target'.
- [ ] Доработать редактор уровня: ходы, цели, блокеры, автоподсчет выполнения.

**Критерий готовности:** дизайнер уровня может собрать и протестировать уровень целиком без ручных правок в коде.

## Этап 3. Бустеры и post-lose сценарии
- [ ] Реализовать бустеры: ракета (вертикаль), лазер (горизонталь), area-burst (4x4 или комбинированный).
- [ ] Добавить UX применения бустеров (кнопка/инвентарь/подсказка).
- [ ] Добавить lose-модалку с предложением докупить ходы/время через rewarded ad.
- [ ] Синхронизировать логику бустеров с целями и препятствиями.

**Критерий готовности:** бустеры влияют на матч в рамках единых правил resolve, без рассинхрона HUD и счета.

## Этап 4. Визуал и оптимизация
- [ ] Перейти на целевую палитру из 5 базовых цветов.
- [ ] Нормализовать размер фишек внутри клетки (80-90%).
- [ ] Добавить фоновые темы уровней (смена каждые 20 уровней).
- [ ] Расширить VFX-эффекты спец-фишек с учетом режима качества.
- [ ] Выполнить профилирование и оптимизацию FPS на больших полях.

**Критерий готовности:** визуальные улучшения не ухудшают стабильность рендера и читаемость поля.

## Этап 5. Meta-слой и удержание
- [ ] Внедрить meta-menu между уровнями.
- [ ] Добавить основного персонажа и нарративные вставки.
- [ ] Подготовить связку персонажа с прогрессом и вовлечением.

**Критерий готовности:** у игрока появляется понятная долгосрочная мотивация, не только "сыграть партию".

## Сопутствующие технические задачи
- [ ] Расширить тестовое покрытие движка и UI-критических флоу.
- [ ] Документировать новые API движка и контракты между React и engine.
- [ ] Актуализировать 'docs/game-backlog.md' и 'docs/README.md' после каждого завершенного этапа.

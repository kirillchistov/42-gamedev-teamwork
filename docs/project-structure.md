# Структура проекта

## Общая схема монорепозитория
Проект организован как 'monorepo' на базе 'lerna':

- 'packages/client' - фронтенд (React + TypeScript + Canvas).
- 'packages/server' - серверная часть (Express/SSR/API).
- 'docs' - проектная и игровая документация.
- 'docker-compose.yml' и 'Dockerfile.*' - локальный/production запуск в контейнерах.

## Клиент ('packages/client')
Ключевые директории:
- 'src/pages' - страницы приложения ('/game', '/login', '/leaderboard' и т.д.).
- 'src/components' - переиспользуемые UI-компоненты.
- 'src/game/match3' - сам модуль игры.
  - 'engine' - ядро, рендер, инпут, эффекты, уровни.
  - 'systems' - рекорды и вспомогательные подсистемы.
  - 'Match3Screen.tsx' - связка движка с React UI.
- 'src/shared/styles' - глобальные стили, темы и переменные.

## Сервер ('packages/server')
Типовой набор:
- HTTP-слой ('Express') и SSR-обвязка;
- роутинг API;
- работа с БД/моделями и middleware;
- сервисы для авторизации, форума, лидерборда (по мере развития).

## Документация ('docs')
В папке 'docs' хранятся:
- требования по игре,
- backlog и roadmap,
- техническое описание движка и визуала,
- гайды по запуску и развитию проекта.

## Потоки данных и зоны ответственности
- **React UI** отвечает за экраны, состояние страницы, модалки, настройки, HUD-обвязку.
- **Canvas engine** отвечает за поле, анимации, матчи, каскады, спец-эффекты.
- **Server/API** отвечает за серверные данные, авторизацию, лидерборд, SSR и persistence.

## Структура маршрутов (высокоуровнево)
- Публичные/общие страницы: лендинг, ошибка, справка.
- Авторизованные зоны: игра, профиль, лидерборд, форум.
- Игровой контур: подготовка -> матч -> результат.

## Скрипты верхнего уровня
- 'yarn bootstrap' - полная установка зависимостей/инициализация монорепо.
- 'yarn dev' - запуск клиентской и серверной части.
- 'yarn dev:client' / 'yarn dev:server' - запуск нужного пакета.
- 'yarn test', 'yarn lint', 'yarn build', 'yarn format'.

## Куда добавлять новый функционал
- Новая игровая механика: 'packages/client/src/game/match3/engine'.
- Новые экраны/настройки игры: 'packages/client/src/pages' + 'Match3Screen'.
- Новые API/бизнес-правила: 'packages/server/src'.
- Новая документация и договоренности: 'docs'.

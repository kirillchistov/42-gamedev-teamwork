## [0:00–0:45] Вступление · 8.10

Привет! Мы — команда 42, делаем игру "3 в ряд" Cosmic Match. Нас 5: Аня, Сергей, Артур, Антон и я, Кирилл, teamlead. Расскажу, что сделали в спринтах **7 и 8**: **Express**, **SSR**, **Ya OAuth**, **лидерборд**, **форум на Postgres**, **темизация** и **авторизация на своём API**.

Стек коротко: React + Redux на клиенте, **Express** отдаёт HTML и проксирует запросы, **Postgres в Docker** — наши данные, **API Практикум** — сессия и лидерборд. В спринтах 5–6 сделали UI и игровой движок; сегодня покажем, как это работает клиент-сервер. Начнем с фундамента — **7.1–7.2**.

## [0:45–2:15] SSR и маршруты · 7.1, 7.2

**Как это устроено:** пользователь открывает URL → **Express** ('server/index.ts') отдаёт не пустой 'index.html', а гонит **React на сервере** ('entry-server.tsx'): Router собирает дерево страницы, Redux делает 'fetchData' для нужных slice’ов, в HTML попадает разметка **и** JSON в 'window.APP_INITIAL_STATE'. Браузер подхватывает ('main.tsx') — это **гидратация**: тот же UI без «мигания», состояние уже есть.

**Показ:** View Source на главной — ищем 'APP_INITIAL_STATE'. '/health' — жив ли наш Node.

**Защита маршрутов:** 'withAuthGuard' на сервере смотрит store: нет пользователя — в HTML уже редирект на логин, а не «пустая форма после F5». Обновите '/forum' без сессии — увидите уход на login.

*Одной фразой:* SSR - это хорошо для SEO + честный первый кадр + данные из Redux.

## [2:15–3:00] OAuth Яндекс · 7.3

**Как это устроено:** кнопка «Войти через Яндекс» ведёт на OAuth с 'redirect_uri' на **наш origin :9000** (whitelist Практикума). После согласия — callback, код меняем на сессию через API Практикума. Браузер ходит на **'/api/v2'** same-origin — **apiProxy** пересылает cookie на 'ya-praktikum.tech', чтобы сессия жила на домене приложения.

**Показ:** '/login' — кнопка и поля. Если env настроен — короткий live-вход; если нет — «вот кнопка, вот 'oauthApi.ts' и маршрут callback, поток тот же».

*Одной фразой:* OAuth без своего IdP — доверяем Практикуму, прокси держит cookie на одном origin.

## [3:00–3:45] Лидерборд · 7.4

**Как это устроено:** 'leaderboardSlice' после логина дергает **'POST /api/v2/leaderboard/cosmicMatch42_bestScore'** — это уже **внешний** API, не наш Postgres. Ответ маппим в таблицу на 'LeaderboardPage': место, ник, очки.

**Показ:** '/leaderboard' — таблица с реальными данными. DevTools → Network — наш POST через прокси, статус 200 с данными.

*Одной фразой:* лидерборд = контракт Практикума + Redux, храним только отображение.

**Показ:** игра → Console: '[Performance] Game session started' и итог по длительности.

## [4:30–5:00] Postgres в Docker · 8.1

**Как это устроено:** 'docker compose' поднимает **postgres:16** на порту **5433** (чтобы не драться с локальным brew на 5432). **Sequelize-миграции** создают таблицы форума и 'ui_theme'. Сервер без БД — 'ECONNREFUSED', поэтому перед демо всегда 'docker compose up -d postgres' и 'yarn db:migrate'.

**Показ:** терминал — 'docker compose ps', одна строка «миграции прошли».

*Одной фразой:* темы и форум живут в Postgres; Практикум отвечает за auth и лидерборд.

---

## [5:00–5:45] Темы UI · 8.3, 8.7

**Как это устроено (клиент):** 'LandingThemeContext' вешает класс на 'body' — 'light-flat', 'light-3d', 'dark-neon'; выбор в **localStorage** и в шапке.

**Как это устроено (сервер):** 'PUT /api/ui/theme' — гость получает cookie 'anonymous_session_id', залогиненный — запись в БД. После входа 'useThemeServerSync' подтягивает тему с сервера, чтобы F5 на другом устройстве не сбрасывал стиль.

**Показ:** переключить тему на '/' или '/game' → Network → 'PUT /api/ui/theme', потом 'GET' при перезагрузке.

*Одной фразой:* тема видна сразу на клиенте, переживает сессию через наш API.

---

## [5:45–6:15] Прокси и auth · 8.4, 8.8

**Как это устроено:** один origin **:9000** — и UI, и API. 'apiProxy.ts' режет трафик: **'/api/v2' → Практикум**, **'/api/forum' и '/api/ui' → наш Node :3000**. Cookie переписываются на наш домен — иначе браузер их не отправит.

На Node **'requirePraktikumAuth'**: 'resolvePraktikumUser' дергает 'GET /auth/user' у Практикума по cookie; нет сессии — **403 Forbidden**, не «пустой JSON».

**Показ:** инкогнито → запрос '/api/forum/topics' → 403 с 'reason: Forbidden'. С логином — 200.

*Одной фразой:* прокси = один домен для фронта; middleware = форум только для своих.

---

## [6:15–8:30] Форум — главный блок · 8.2, 8.5, 8.6, 8.9

**Как это устроено (бэкенд):** 'forumRouter' — REST: топики, дерево комментариев, CRUD, модераторы по whitelist id. Реакции — отдельная таблица, **UNIQUE** на (комментарий, пользователь, эмодзи), whitelist emoji в 'forumEmojiGuard'.

**Как это устроено (клиент):** 'forumApi' + thunks в 'forumSlice'. Список тем показывает два счётчика: **Комментарии** (корневые) и **Ответы** (вложенные) — чтобы с первого экрана было видно активность. На странице темы — 'ForumCommentReactions': picker 🙂, чипы, toggle без перезагрузки страницы.

**Показ по шагам (не торопиться):**

1. '/forum' — список после логина.  
2. «+ Новая тема» — POST, тема в списке.  
3. Открыть тему — текст + лента.  
4. Комментарий → счётчик **Комментарии** +1.  
5. «Ответить» → вложенный блок, **Ответы** +1.  
6. 🙂 → эмодзи-чип; повтор — снять реакцию.  
7. Выйти / инкогнито — редирект или 403 (связка с 8.4).

*Одной фразой:* форум — наш Postgres + наш API; UX — дерево, счётчики, реакции в один клик.

## Шпаргалка: что сказать, если спросят «как именно работает X»

| Компонент | Ответ в одно предложение |
| --- | --- |
| 'entry-server.tsx' | Собирает HTML + 'APP_INITIAL_STATE' под конкретный URL до ответа браузеру. |
| 'withAuthGuard' | На SSR не пускает в защищённый route без user в store. |
| 'apiProxy.ts' | Same-origin: '/api/v2' наружу, '/api/forum' и '/api/ui' на наш Node. |
| 'requirePraktikumAuth' | Проверяет cookie через Практикум; без сессии — 403. |
| 'forumRouter' | CRUD топиков/комментариев + реакции в Postgres. |
| 'forumSlice' | Redux: список, тема, optimistic toggle реакций, редирект на login при 403. |
| 'LandingThemeContext' | Класс на 'body' + sync с 'PUT/GET /api/ui/theme'. |
| 'performanceMetrics' | Mark/measure + long tasks в консоль на время сессии игры. |

## Советы темпа (чтобы уложиться в 12 минут)

- **Не расширять** OAuth и Pixi — только схема + кнопка / один кадр игры.  
- **Форум** — единственный блок, где можно «задержаться»; остальное — 1–2 действия на экране.  
- **7.6 и 8.1** — по 15–30 с, иначе съедят форум.  
- Финальный чеклист — **таблица вслух за 60–90 с**, не пересказ каждой задачи.  
- Держать **9000** в адресной строке; если видите «голый» API на 3000 — вы не на SSR-клиенте.

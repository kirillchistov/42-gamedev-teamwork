# Cosmic Match-3: План доработок в 6 спринте

## 6.1.1 Улучшить ядро игры: отделить логику от визуала: 
[x] Что: состояние поля,  матчи, каскады и пр. — отдельно; canvas-отрисовка и анимации — отдельно.
[x] Как: в engine/core/* оставить только чистые функции без canvas/window (board ops, match detect, scoring, special rules);
в engine/bootstrap.ts оставить оркестрацию партии (фазы, таймер, input, HUD);
в engine/renderer.ts держать только отрисовку и визуальные эффекты;
ввести явные типы GameState, ResolveStep, GameOptions, чтобы bootstrap не хранил “магические” поля.
[x] Результат: проще развивать и дебажить игру; core можно тестировать отдельно.
###  Статус: in review

## 6.1.6 Спец-фишки и препятствия: создание и активация
[x] Что:  спец-фишки: “зачистка ряда” + “бомба (3x3)”; анимационные фазы highlight → clear → fall → refill.
[x] Как: расширить тип клетки (kind, specialType?, blocked? для будущих препятствий);
генерация спец-фишки при 4+/сложной форме; активация при участии спец-фишки в матче или прямом свапе; в renderer добавить простые эффекты вспышки/радиуса без изменения внешних страниц.
[x] Результат: Игрок получает доп фан со спецэффектами; Добавить простые визуальные эффекты для спец-фишек.
###  Статус: in review

## 6.1.7 Поиск возможных ходов + пересдача при тупике
[x] Что: Проверка “есть ли ход”; если нет — автоматический shuffle.
[x] Как: добавить findPossibleMoves(board) (перебор соседних свапов + проверка матча);
после каждого resolve проверять наличие ходов; при пустом списке запускать анимированный shuffleBoard() до валидного состояния без стартовых матчей.
[x] Результат: Игрок не застревает на “тупиковом” поле.
###  Статус: in review
